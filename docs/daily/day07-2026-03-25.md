## 이슈/해결
- **MySQL 연결 오류 장기화**
  - 원인 최종 정리: JDBC 드라이버가 `127.0.0.1` → `localhost` 변환, MySQL 내부에 `learnhub@localhost` / `learnhub@%` / `learnhub@127.0.0.1` 계정이 혼재하며 각각 비밀번호 해시가 달라 인증 실패
  - 해결 방향: 다음 작업 시 컨테이너 완전 재생성 후 `MYSQL_USER=learnhub`, `MYSQL_PASSWORD=learnhub1234` 환경변수로 단일 계정만 생성하여 혼재 방지

## 학습 포인트
- Docker MySQL 컨테이너에서 `MYSQL_USER` / `MYSQL_PASSWORD` 환경변수로 생성된 계정은 `@%` 호스트로만 생성됨. 이후 수동으로 `@localhost`, `@127.0.0.1` 계정을 추가하면 비밀번호 해시가 달라 충돌 발생
- 가장 안전한 방법: 컨테이너 최초 생성 시 환경변수로만 계정 설정하고 이후 수동 계정 추가 금지
- `mysql_native_password` 플러그인은 MySQL 8.4부터 deprecated 예정이므로 장기적으로는 `caching_sha2_password` + JDBC `allowPublicKeyRetrieval=true` 조합 사용 권장
