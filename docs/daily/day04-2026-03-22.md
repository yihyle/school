- LectureProgress(학습 진도) API 구현 - 진도 저장 / 강의 완료 처리 / 강의별 진도율 조회

## 이슈/해결
- **LectureProgress 진도율 계산 시 N+1 문제 가능성**
  - 강의별 진도 조회 시 Section → Lecture → Progress 순으로 조회하는 구조
  - 현재는 `findByUserIdAndLectureIdIn` 쿼리로 한 번에 조회하도록 처리

## 학습 포인트
- JPA `@UniqueConstraint`로 enrollment(user_id, course_id), lecture_progress(user_id, lecture_id) 복합 유니크 제약 조건 적용
- `orElseGet`으로 진도 레코드가 없을 때 자동 생성하는 upsert 패턴 구현
- 진도율 계산: 완료된 강의 수 / 전체 강의 수 × 100, 소수점 2자리 반올림
