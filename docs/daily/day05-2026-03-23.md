## 이슈/해결
- **MySQL 연결 오류: Access denied for user 'learnhub'@'localhost'**
  - 원인: MySQL JDBC 드라이버가 JDBC URL의 `127.0.0.1`을 내부적으로 `localhost`로 변환하여 연결 시도. `learnhub@localhost` 계정의 비밀번호가 불일치
  - 시도한 해결 방법
    1. application.yml URL을 `localhost` → `127.0.0.1`로 변경 → 동일 오류
    2. Colima를 aarch64로 재생성 후 MySQL 컨테이너 재시작 → 동일 오류
    3. `--skip-name-resolve` 옵션으로 MySQL 컨테이너 재생성 → 동일 오류
    4. `learnhub@localhost` 계정 `mysql_native_password`로 변경 → 미해결 (다음날 계속)

## 학습 포인트
- MySQL JDBC 드라이버는 `127.0.0.1`을 `localhost`로 resolve하는 동작이 있어 `@localhost`와 `@127.0.0.1` 계정을 별도로 관리해야 함
- `--skip-name-resolve` 옵션은 MySQL 서버의 DNS 역방향 조회를 비활성화하지만 JDBC 드라이버 자체의 변환은 막지 못함
- `mysql_native_password` vs `caching_sha2_password`: MySQL 8.0 기본 인증 방식은 `caching_sha2_password`이며 JDBC 드라이버 버전에 따라 호환성 문제 발생 가능
