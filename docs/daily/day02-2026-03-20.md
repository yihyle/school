## 이슈/해결
- **MySQL 컨테이너 반복 종료 (Fatal glibc error: CPU does not support x86-64-v2)**
  - 원인: Colima VM이 x86_64 모드로 실행 중이었으나 해당 CPU가 x86-64-v2 명령어셋 미지원
  - 해결: `colima delete --force` 후 `colima start --arch aarch64`로 VM 재생성, docker info에서 `Architecture: aarch64` 확인 후 MySQL 정상 실행

- **ARM 이미지 플랫폼 불일치 경고**
  - 원인: `arm64v8/mysql:8.0` 이미지가 aarch64 Colima에서 manifest 없음 오류 발생
  - 해결: 플랫폼 명시 없이 `mysql:8.0` 사용 시 aarch64 환경에서 자동으로 ARM 이미지 pull됨, docker-compose.yml에 `platform: linux/aarch64` 명시

## 학습 포인트
- Colima는 기본적으로 x86_64 VM으로 시작되므로 Apple Silicon(M1/M2/M3) 환경에서는 `--arch aarch64` 옵션으로 명시적으로 생성해야 ARM 네이티브 이미지 실행 가능
- Flyway 마이그레이션 파일명 규칙: `V{버전}__{설명}.sql` (언더스코어 2개)
- enrollment, lecture_progress 테이블에 복합 UNIQUE 제약 조건 추가로 중복 수강/진도 방지
