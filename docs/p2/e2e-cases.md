## 1. 인증 시나리오

### TC-AUTH-01 회원가입 성공

| 항목 | 내용 |
|---|---|
| 전제 조건 | 가입되지 않은 이메일 |
| 단계 | 1. /signup 접속 → 2. 이메일/비밀번호/닉네임 입력 → 3. 학생 역할 선택 → 4. 가입 버튼 클릭 |
| 기대 결과 | 메인 페이지 리다이렉트, 헤더에 닉네임 표시 |
| 자동화 파일 | `e2e/auth.spec.ts` |

### TC-AUTH-02 로그인 성공 → 보호 라우트 접근

| 항목 | 내용 |
|---|---|
| 전제 조건 | 가입된 학생 계정 존재 |
| 단계 | 1. /login 접속 → 2. 이메일/비밀번호 입력 → 3. 로그인 → 4. /dashboard 접속 |
| 기대 결과 | 대시보드 정상 표시 (리다이렉트 없음) |
| 자동화 파일 | `e2e/auth.spec.ts` ✅ 구현됨 |
