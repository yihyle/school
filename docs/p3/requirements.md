# LearnHub P3 요구사항 명세서

> P3 단계: 운영형 / 외부연동 + 관측성
> 구독·알림(디스코드) · 스케줄러/배치 · 결제 · 모니터링 · 보안 · CI/CD

---

## 1. 목표

P2까지 완성된 인증·강의·커뮤니티 위에 **운영에 필요한 외부연동과 관측성**을 더한다.

- 사용자가 사이트에 접속하지 않아도 **외부 채널(디스코드)** 로 소식을 받는다.
- **스케줄러/배치**로 정해진 시간에 데이터를 처리한다(미학습 리마인드, 알림 발송).
- **결제**로 유료 강의를 구매하면 자동으로 수강 등록된다.
- **헬스체크/구조화 로그**로 운영 중 문제를 발견할 수 있다.
- **CI/CD**로 빌드·테스트·배포를 자동화한다.

## 3. 비기능 요구사항

- **신뢰성**: 외부 연동(디스코드/PG) 실패가 핵심 트랜잭션을 막지 않는다 → 아웃박스(Outbox) 패턴으로 분리.
- **멱등성**: 알림·결제·배치 모두 고유 키로 중복 실행을 방지한다.
- **관측성**: 모든 요청 로그에 traceId, 헬스체크/메트릭 노출.
- **보안**: 결제·구독은 인증 필요, 헬스체크 외 actuator는 ADMIN 한정, 결제 금액 위변조 검증, JPA 파라미터 바인딩(SQLi 방어), 출력 이스케이프(XSS 방어).

## 4. 범위 밖 (Out of scope)

- 환불/부분취소, 정기결제(구독 결제)
- 실제 이메일 발송(SMTP) — 본 단계는 디스코드 Webhook 채널만 구현
- 분산 스케줄러(ShedLock 등) — 단일 인스턴스 가정
